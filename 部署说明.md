# 部署说明

## Vercel 环境变量

只在 Vercel 项目设置中添加：

```text
API_FOOTBALL_KEY=你的 API-Sports Key
```

不要把 Key 写入 `src`、GitHub、截图或小程序代码。前端只请求网站自己的
`/api/worldcup`，真实 Key 仅由 Vercel 服务端读取。

## 数据更新与额度保护

- 页面打开后立即同步，页面可见时约每 60 秒刷新一次。
- `/api/worldcup` 使用 Vercel CDN 和服务端内存缓存，同一分钟内的访客共享数据。
- 任意查询参数会重定向到固定地址，避免利用随机参数绕过 CDN 缓存。
- 旧 `/api/football` 仅允许 `fixtures` 和 `standings`，并固定为 2026 世界杯。
- 接口失败时会尽量返回最近一次有效数据，页面不会立刻变空。

建议再在 Vercel Firewall 中给 `/api/*` 设置合理的 IP Rate Limit，作为额外防护。

## 发布

把项目覆盖到 GitHub 仓库后提交：

```bash
git add .
git commit -m "connect live World Cup data"
git push origin main
```

Vercel 自动部署完成后，打开 `/api/worldcup`，确认能看到 `fixtures` 和
`standings`，然后强制刷新网站。
